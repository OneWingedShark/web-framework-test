Mneson Conventions

(C) Marius Amado Alves (amado.alves@netcabo.pt, maa@liacc.up.pt)
License: SCD Conditions (www.softdevelcoop.org)
Version of this document: 20040407b

The Mneson Conventions define a way to represent complex structures, notably attributes and typed links, in the untyped graph.

Basic attribute representation convention

1. For each attribute type, there exists a vertex that represents it.

2. Each target of an attribute type is an attribute instance.

3. An attribute instance has one or more sources.

4. Of necessity, one is the attribute type. Each of the others is a subject of the attribute.

5. The targets of an attribute instance represent its value.

Alternate valueless attribute instance representation convention

6. An atribute type represents a valueless attribute instance.

Simple attribute type naming convention

7. A string s is an attribute type of name s.

Simple typed connection representation convention

8. An attribute instance represents a typed connection.

Simple set representation convention

9. For a vertex that represents a set, its targets represent its elements.

Simple record representation convention

10. A record is a set of attribute instances, the components of the record.

Simple vector representation convention

11. A vector is a one dimensional array of integer index.

12. A vector of index range [First, Last] is represented as a record whose components are of type First to Last.

Why attribute instances

And not attribute values directly represented in their place.

Because if they were, then the attribute types of distinct subjects with the same value could not be distinguished.
